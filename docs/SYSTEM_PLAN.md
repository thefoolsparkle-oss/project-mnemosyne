# 长期系统计划

这个项目的目标不是维护一堆固定人格文件，而是让每个用户都能创建、拥有、持续完善自己的对话对象。

系统长期形态：

- 用户注册、登录，并拥有自己的资料页。
- 用户通过选项和自由描述创建专属人格。
- 人格不是一次性 prompt，而是会随着聊天、记忆和用户偏好持续进化。
- 聊天界面接近 QQ / 微信：头像、昵称、会话列表、左右气泡、资料卡。
- 记忆系统独立于 LLM 上下文窗口，不依赖模型“自己记住”。
- 外貌、头像和角色形象后续由专门模块生成和维护。

## 产品入口

首次进入时，系统不要让用户直接面对一个大文本框，而是问：

> 你想和什么样的人交谈？

用户可以从少量选项开始，也可以随时用自然语言补充。

选项不能太多，一屏只放关键维度：

- 氛围：温柔陪伴、冷静克制、活泼吐槽、成熟可靠、安静亲密
- 关系：朋友、恋人、暧昧、姐姐/哥哥感、学习伙伴、倾听者
- 说话方式：短句、话多一点、少追问、会主动关心、可以吐槽、少用表情
- 边界：现实感强、二次元风格、不要太黏人、不要说教

用户选到一半也可以直接生成。生成后仍然可以继续完善。

## 内部 AI 模块

这些不是一开始就必须接不同模型。第一版可以共用同一个 LLM，只是使用不同 prompt 和不同数据输入。以后需要扩展时，再拆成独立服务。

### 人格生成器（Forge）

负责根据用户选择和描述生成初始人格。

输入：

- 用户选择的标签
- 用户自由描述
- 用户资料
- 安全边界

输出：

- 人格名称
- 关系定位
- 性格标签
- 说话风格
- 禁忌和边界
- 首次问候风格
- 可用于主聊天的 system prompt

### 记忆档案员（Archivist）

负责从聊天中提取、分类、去重和总结长期记忆。

它不直接聊天，只在后台工作。

记忆类别：

- user_profile：用户稳定资料，如昵称、生日、偏好
- preference：长期喜好和讨厌
- relationship：用户希望的关系、称呼、距离感
- boundary：不能触碰的称呼、话题、语气
- event：发生过的重要事件
- emotional_pattern：用户常见情绪模式
- persona_feedback：用户对人格表现的反馈

### 用户画像器（Mirror）

负责整理用户的聊天逻辑、性格倾向和互动偏好。

它关注的是“用户怎样被回应会更舒服”：

- 喜欢长回复还是短回复
- 喜欢主动关心还是低打扰
- 讨厌什么称呼
- 情绪低落时需要陪伴还是建议
- 是否容易被说教感刺激

Mirror 的输出会影响人格微调，但不能直接覆盖用户明确设置。

### 人格调整器（Sculptor）

负责根据长期反馈持续完善人格。

它不是每条消息都改人格，而是在积累足够证据后生成“人格修订建议”。

例如：

- 用户多次表示不喜欢太黏：降低依赖感表达。
- 用户喜欢短句：压缩回复长度。
- 用户喜欢角色主动一点：增加轻微主动关心。
- 用户讨厌某个称呼：写入硬边界。

所有人格变更都要保留版本记录，允许回滚。

### 形象生成器（Visage）

负责角色外貌、头像和图像生成。

输入：

- 用户描述的外貌
- 人格气质
- 聊天中逐渐补充的形象细节

输出：

- 外貌设定
- 图片生成 prompt
- 头像资源
- 角色资料卡视觉描述

这部分可以晚一点做，不能挡住聊天和记忆主线。

### 主聊天 AI（Core）

真正直接和用户说话的模块。

Core 每次回复时接收：

- 当前人格版本
- 用户资料
- 最近聊天记录
- 召回的长期记忆
- 当前关系状态
- 当前情绪状态
- 安全边界

Core 不负责长期总结，不负责自己改人格。它只负责在给定上下文里稳定地对话。

## 记忆系统设计

LLM 的上下文窗口不是记忆。真正的记忆必须落到数据库。

每条消息进来后，系统要做四层处理：

1. 写入原始消息表。
2. Archivist 提取候选记忆。
3. 对候选记忆打分、分类、去重。
4. 在未来对话中按相关性召回。

### 记忆分层

短期记忆：

- 最近 N 条对话。
- 用于保持当前话题连续。
- 可以直接塞进上下文。

情节记忆：

- 某天发生过的具体事情。
- 例如“用户今天考试很累”。
- 会逐渐过期或被总结。

语义记忆：

- 稳定事实。
- 例如“用户喜欢短句”“用户讨厌被叫某个称呼”。
- 长期保留，优先级高。

关系记忆：

- 用户和人格之间的关系设定、称呼、边界、互动习惯。
- 比普通偏好更重要。

人格记忆：

- 这个 AI 自己被用户塑造成什么样。
- 例如“少说教”“主动一点”“不要太甜”。
- 由 Sculptor 管理版本。

### 记忆召回

每次回复前，不是把所有记忆塞给模型，而是取少量最相关的。

召回顺序：

1. 硬边界和称呼禁忌
2. 当前人格核心设定
3. 与本轮消息相关的长期记忆
4. 最近对话
5. 用户画像摘要

每条记忆至少要有：

- id
- user_id
- persona_id
- type
- text
- importance
- confidence
- source_message_id
- created_at
- updated_at
- last_used_at
- archived

### 记忆抗遗忘

为了避免聊长了失忆，不能只依赖最近对话。

需要三个摘要：

- conversation_summary：当前会话到目前为止发生了什么
- user_profile_summary：用户稳定画像
- persona_summary：当前人格版本摘要

这些摘要定期更新，并在每次聊天时进入上下文。

### 记忆抗污染

用户偶尔一句玩笑不能立刻变成长期记忆。

需要规则：

- 重要称呼、边界、强偏好可以立即写入。
- 模糊倾向先写成低置信度候选。
- 多次出现后提高置信度。
- 与旧记忆冲突时不覆盖，先标记 conflict。
- 由后续确认或更多证据解决冲突。

## 数据模型草案

长期应该从 JSON 文件迁到数据库。

第一阶段可以用 SQLite，后续再换 PostgreSQL。

核心表：

- users：账号、密码哈希、状态
- user_profiles：头像、昵称、生日、签名、简介
- personas：用户创建的人格当前版本
- persona_versions：人格历史版本
- conversations：会话
- messages：消息
- memories：长期记忆
- memory_links：记忆与消息、人格版本、会话的关联
- avatar_assets：头像和形象资源

## 开发阶段

### Phase 1：地基

- 引入数据库。
- 建立用户注册、登录、退出。
- 建立用户资料页。
- 移除对固定 `personas/evelyn`、`personas/yuzu` 的核心依赖。
- 增加人格创建向导的数据结构。

完成标志：

- 每个用户能登录。
- 每个用户能创建至少一个专属人格。
- 聊天使用数据库中的人格，而不是本地 persona 文件。

### Phase 2：聊天系统

- 改成 QQ / 微信式界面。
- 增加会话列表。
- 每个会话绑定一个 persona。
- 消息全部写入数据库。
- 每个用户、每个人格、每个会话互相隔离。

完成标志：

- 多个用户同时使用不会共享人格、记忆或聊天历史。

### Phase 3：记忆系统

- Archivist 提取长期记忆。
- Mirror 生成用户画像摘要。
- 记忆召回进入主聊天上下文。
- 增加记忆去重、重要度、置信度、冲突标记。

当前进度：

- 已新增数据库记忆表接入主聊天链路。
- 已新增 `Archivist`，支持 LLM 提取和本地规则兜底。
- 已新增 `Mirror` 的第一层用户互动洞察。
- 已新增 `/api/memories` 和 `/api/insights` 调试接口。
- 已新增分层记忆表：`memory_events`、`memory_episodes`、`memory_facts`、`memory_relations`、`memory_summaries`、`memory_links`。
- 已新增记忆编号：`EVT-*`、`EP-*`、`FACT-*`、`REL-*`。
- 已新增 `/api/memory/layered` 调试接口。
- 详细设计见 `docs/MEMORY_ARCHITECTURE.md`。

完成标志：

- 长对话后仍能稳定记住称呼、偏好、边界和关系。

### Phase 4：人格进化

- Sculptor 定期生成人格修订建议。
- 人格版本化。
- 用户可以查看、接受、撤销人格变化。

完成标志：

- 人格会适应用户，但不会无控制地漂移。

### Phase 5：形象系统

- Visage 维护外貌设定。
- 生成头像 prompt。
- 接入图片生成。
- 聊天界面显示用户头像和 AI 头像。

完成标志：

- 每个人格有自己的资料卡、头像和外貌设定。

## 当前第一步

下一步应该先做 Phase 1 的最小闭环：

1. 增加 SQLite 数据库。
2. 增加用户表和登录注册接口。
3. 增加 persona 表。
4. 增加人格创建向导接口。
5. 让聊天读取数据库 persona。

暂时不要先做图片，也不要先做复杂人格进化。

原因：

- 没有账号，记忆一定会混。
- 没有数据库，长期记忆一定不可维护。
- 没有 persona 表，后续 Forge / Sculptor / Visage 都没有稳定落点。
